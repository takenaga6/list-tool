# 企業リストアップツール 仕組みドキュメント

## 概要

offi-stretch.com（オフィスストレッチサービス）の営業向けに、
**B2B企業を自動でリストアップ → HubSpot CRMに登録**するツール。

---

## 全体フロー

```
キーワード × 期間
    ↓
① DuckDuckGo検索（search_agent）
    ↓
② 企業サイトをスクレイピング（scraper_agent）
    ↓
③ 企業HP判定（株式会社かどうか確認）
    ↓
④ ランク判定 A/B/C/NG（rank_agent）
    ↓
⑤ HubSpot CRMに自動登録（hubspot_agent）
    ↓
⑥ クエリ学習（keyword_agent）
```

---

## 各エージェントの役割

### search_agent.py　―　DuckDuckGo検索
- キーワードと期間を指定してDuckDuckGo検索
- Googleはキャプチャで弾かれるためDuckDuckGoを使用
- **除外**: SNS・求人・Wikipedia等は最初から除外
- 検索結果を `{url, title, snippet, search_query}` 形式で返す

---

### scraper_agent.py　―　スクレイピング＋企業HP判定

#### スクレイピング対象ページ
- トップページ
- `/company` `/about` `/corporate` `/gaiyou` 等の会社概要ページ（最大4ページ）

#### 企業HP判定（check_company_fields）
以下5項目のうち **3つ以上**あれば企業HPと判定。

| 項目 | 判定内容 |
|------|---------|
| 法人格 | 株式会社 / 合同会社 / 有限会社 の記載 |
| 代表者 | 代表取締役 / 代表者 / CEO 等の記載 |
| 事業内容 | 事業内容 / 会社概要 / Company / About Us 等 |
| 住所 | 〒郵便番号 または 都道府県＋市区町村 |
| TEL | TEL / 電話番号 / 0X-XXXX-XXXX 形式 |

#### 除外ルール

| 除外条件 | 理由 |
|----------|------|
| 「運営会社」の記載あり | SaaS・プロダクトのサービスサイト |
| `.go.jp` `.or.jp` `.ac.jp` `.lg.jp` | 官公庁・団体・大学（株式会社ではない） |
| 上記チェックが通らない | 不明なURL → company_url を空にする |

> 企業URLが不明な場合はHubSpotに**URLを登録しない**（空白のまま）

#### 媒体記事URLの取得（find_media_article_url）
- 媒体クエリ（KENJA GLOBAL等）で見つかった企業の「掲載記事URL」を別途検索
- HubSpotの「説明」欄に記載するために使用
- 例: `kenja.tv/articles/xxxx` のような記事URLを取得

#### 取得する企業情報
- 会社名・企業URL・代表者名
- 電話番号・郵便番号・都道府県・住所
- 従業員数・業種

### rank_agent.py　―　A/B/C/NGランク判定

#### ターゲット企業イメージ
GMO・FBモーゲージ・グランドバリュー・マブチS&T・プラグマのような企業

#### スコアリング（最大12点）

| シグナル | 点数 | 内容 |
|----------|------|------|
| PR媒体掲載 | +2 | KENJA GLOBAL・社長名鑑等の社長インタビュー媒体に掲載 |
| 健康経営 | +2 | 健康経営優良法人・健康経営宣言等 |
| CEO健康意識 | +1 | 社長のランニング・マラソン・登山等の趣味 |
| 高利益率業種 | +2 | IT/金融/人材/コンサル/士業 等 |
| 既存福利充実 | +1 | 食事補助・リモートワーク・社員旅行等あり |
| フィジカルケア未着手 | +1 | 福利厚生はあるがマッサージ・ストレッチがない（提案余地大） |
| 社員数10〜200名 | +1 | ターゲット規模 |
| 非上場 | +1 | 上場企業は除外 |
| オフィス勤務 | +1 | 在宅・リモートより対面勤務が多い |

#### ランク基準

| ランク | スコア | 意味 |
|--------|--------|------|
| **A** | 6点以上 | 即アプローチ推奨 |
| **B** | 4〜5点 | 候補 |
| **C** | 2〜3点 | 一応リストアップ |
| **NG** | 1点以下 | 登録しない |

### hubspot_agent.py　―　HubSpot CRM登録

#### 登録フィールド

| HubSpotフィールド | 内容 |
|------------------|------|
| 会社名 | 企業名 |
| ウェブサイト | 企業URL（株式会社確認済みのみ） |
| 電話番号 | TEL |
| 住所・郵便番号・都道府県 | 所在地情報 |
| 従業員数 | スクレイプで取得した人数 |
| 業種 | 推定業種 |
| **説明欄** | 媒体記事URL ＋ 検索クエリ ＋ ランク ＋ 理由 |

#### 説明欄の記載例
```
媒体記事URL: https://kenja.tv/articles/xxxxx | 検索クエリ: KENJA GLOBAL 株式会社 | ランク: A(8点) | 理由: PR媒体掲載, 健康経営, IT企業
```

#### 重複チェック
- 会社名またはドメインが既にHubSpotに存在する場合はスキップ
- 同セッション内でも同一ドメインは2度登録しない

---

### keyword_agent.py　―　クエリ自動生成＋学習

#### クエリグループ（合計264種類）

| グループ | 内容 | 例 |
|----------|------|-----|
| PR媒体 | 社長インタビュー媒体名 | KENJA GLOBAL 株式会社 |
| 健康福利 | 健康経営・福利厚生系 | 健康経営優良法人 IT企業 |
| CEO健康 | 社長の趣味・健康活動 | 代表取締役 マラソン 会社 |
| 業種×健康 | 業種と健康の掛け合わせ | IT企業 社員食堂 株式会社 |
| 地域×健康 | 地域と健康経営 | 東京 健康経営 中小企業 |
| 成長企業 | 急成長・資金調達等 | シリーズA 資金調達 スタートアップ |

#### キーワード学習（NG率によるデモーション）
- **3回以上実行**かつ **NG率90%以上**のクエリは優先度を下げる
- A/Bランク実績のあるクエリは優先度を上げる
- 実行順: カスタム → A実績 → AB率高 → 未実行 → 高NGクエリ

## 検索期間（複数選択可）

| 番号 | 期間 |
|------|------|
| 1 | 過去1週間 |
| 2 | 過去1ヶ月 |
| 3 | 過去3ヶ月 |
| 4 | 過去6ヶ月 |
| 5 | 過去1年 |
| 6 | 期間指定なし |

複数選択例: `1,2,3` → 1週間・1ヶ月・3ヶ月を全キーワードで検索

## ファイル構成

```
list_tool/
├── main.py                  # メイン実行ファイル
├── config.py                # 設定（APIトークン・定数）
├── agents/
│   ├── search_agent.py      # DuckDuckGo検索
│   ├── scraper_agent.py     # スクレイピング・企業HP判定
│   ├── rank_agent.py        # A/B/C/NGランク判定
│   ├── hubspot_agent.py     # HubSpot登録
│   └── keyword_agent.py     # クエリ自動生成・学習
└── output/
    ├── results.csv          # 登録結果CSV
    ├── ng_list.csv          # NGリスト
    └── list_tool.log        # 実行ログ
```

---

## 実行方法

```cmd
cd C:\Users\user\list_tool
python main.py
```

対話形式で以下を入力：
1. 検索期間（複数可、例: `1,3`）
2. 検索モード（`1`=自動推奨 / `2`=手動）
3. 目標登録件数（デフォルト50件）
