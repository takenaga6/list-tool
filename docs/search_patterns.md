# 有効検索パターン・知見

**ファイル**: `agents/keyword_agent.py` — ヒット率・ランク率を自動学習
**ログ**: `output/keyword_stats.json`

---

## 高効率パターン（実績ベース）

### カテゴリA: 媒体名クエリ（最高効率）

媒体名を含むクエリは PR媒体クエリ経由ボーナス +2点が付く。
スクレイプ不要で掲載確定のため最も精度が高い。

```
KENJA GLOBAL 株式会社
賢者グローバル 掲載企業
SUPER CEO 掲載 IT企業
B-PLUS インタビュー 人材
ベンチャー通信 掲載 コンサル
社長名鑑 掲載
経営者プライム 健康経営
```

### カテゴリB: 健康経営キーワード

```
健康経営優良法人 IT 東京
健康経営優良法人 コンサルティング
ウェルビーイング 取り組み 株式会社
健康経営宣言 人材サービス
ブライト500 認定 東京
```

### カテゴリC: 福利厚生 + 業種

```
法定外福利厚生 充実 IT 東京 株式会社
法人 マッサージ 導入 コンサル
社員 健康 取り組み 情報通信
福利厚生 充実 社労士 事務所
```

### カテゴリD: PR/広告投資シグナル

```
社長 インタビュー 健康 IT企業
代表取締役 メッセージ 健康経営
アワード受賞 SaaS 東京
メディア掲載 人材会社 健康
```

## 効果が低いパターン（避けるべき）

| パターン | 問題点 |
|---|---|
| 「株式会社 東京」のみ | ノイズが多い |
| 「健康 サービス」のみ | 医療・toC企業が混入 |
| 「IT 企業 採用」 | 求人サイトが上位に来る |
| 地名のみ + 業種 | 大企業・チェーン店が混入 |

## 検索クールダウン設定

Google検索のブロック防止:
- `SEARCH_DELAY_MIN = 3` 秒
- `SEARCH_DELAY_MAX = 5` 秒

DuckDuckGo: 上記より短く設定済み（1〜3秒）。

## keyword_stats.json の見方

```json
{
  "KENJA GLOBAL 株式会社": {
    "total": 50,      // 検索件数
    "hit": 30,        // 企業HP特定できた件数
    "ranked": 20,     // スコア3点以上だった件数
    "hit_rate": 0.60, // ヒット率
    "rank_rate": 0.40 // ランク率（全検索結果に対して）
  }
}
```

`hit_rate` × `rank_rate` が高いパターンを優先的に使う。

## 検索パターンを追加する方法

`agents/keyword_agent.py` の `KEYWORD_PATTERNS` リストに追記。
フォーマット: `[キーワード1, キーワード2, ...]`

実行後、`keyword_stats.json` に成績が自動記録される。
次回の自動モードでヒット率上位が優先提案される。

## Google 403ブロック対策

DuckDuckGoを主検索エンジンとして使用（Google403対策済み）。
ただし DuckDuckGo もレートリミットがあるため:
- daemonモード時は `--interval=60`（デフォルト60分）で間隔を確保
- MAX_WORKERS を減らすとブロックされにくくなる（デフォルト5）

設定場所: `config.py` — `MAX_WORKERS` / `SEARCH_DELAY_MIN/MAX`
