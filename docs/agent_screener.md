# Agent1: スクリーナー（pre_screen）詳細

**ファイル**: `agents/rank_agent.py` — `pre_screen()` 関数
**タイミング**: スクレイピング前（HTTPリクエスト発生なし・高速）

---

## 役割

検索結果のスニペット・URL・タイトルだけを見て、スクレイピングの必要がないNGをはじく。
余計なHTTPリクエストを発生させずに処理を高速化する「フィルター1段目」。

---

## チェック項目（順番に適用）

### ① 上場・大規模企業チェック
| パターン | NG理由 |
|---|---|
| 東証・上場企業・証券コード・プライム市場等 | 上場企業 |
| タイトルに「ホールディングス / Holdings」 | ホールディングス（大規模企業） |
| タイトルに「GROUP・グループ会社・グループ子会社」 | 大手グループ企業 |

### ② 従業員数チェック（スニペット記載時）
| 条件 | NG理由 |
|---|---|
| 従業員数 > 200名 | 従業員N名（200名超） |
| 従業員数 < 10名 | 従業員N名（規模不足） |

**抽出パターン**: `従業員[数人]?\s*[：:約]?\s*(\d+)\s*名?`

### ③ 多拠点チェック
以下のいずれかにマッチした場合NG:
- `[2-9]拠点以上`
- `全国\d+拠点`
- `全国各地`
- `支社/支店/営業所 + 数字`
- `全国展開 / 全国ネットワーク`

### ④ NG業種チェック（テキスト先頭300文字）
`config.py` の `NG_INDUSTRY_KEYWORDS` を参照（[NG条件詳細](ng_conditions.md)）

---

## 戻り値

```python
(passed: bool, ng_reason: str)
```

- `passed=True` → スクレイピングに進む
- `passed=False` → スキップ（`ng_reason` をログに記録）

---

## 精度改善のポイント

- スニペットに従業員数が載っていない場合はスルーして scraper でチェック
- NG業種は `config.py` の `NG_INDUSTRY_KEYWORDS` を編集して追加可能
- 「支店が1つだけ」の企業が誤除外される場合は `_MULTI_BRANCH_PATTERNS` を調整
