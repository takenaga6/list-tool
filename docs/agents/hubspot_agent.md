# Agent: HubSpotエージェント（hubspot_agent.py）

**目的**
- 収集・抽出された企業データをHubSpot CRMに登録（Company + Contact）する。
- すでに登録済みの企業を重複チェックし、重複なら登録をスキップ。
- 登録に失敗した企業はNGリストに記録し、以降のターゲットから除外する。

---

## 1. 役割（責務）

### ✅ やること
- 企業の重複チェック（ドメイン / 会社名）
- 企業（Company）登録
- 代表者（Contact）登録と企業への紐付け
- 登録失敗時の処理（ログ出力 / NGリスト追記 / 再試行ロジック）

### ❌ やらないこと
- 企業データの収集・スクレイピング（scraper_agent が担当）
- ランク付け・候補選定（rank_agent が担当）
- 企業の除外判断（search_agent/hubspot_auditor が担当）

---

## 2. 入力（company_data）

hubspot_agent は `company_data: dict` を受け取り、下記キーを参照する。

| キー | 内容 | 備考 |
|---|---|---|
| `company_name` | 会社名 | 必須（ない場合は空文字として扱う） |
| `company_url` | 企業ホームページ | 登録時 `website`/`domain` に使用 |
| `phone` | 電話番号 | 正規化して `+81...` 形式にして送信（失敗時は空にして再送） |
| `address` | 住所 | `address` に送信 |
| `prefecture` | 都道府県 | `state`（都道府県）および `city` に同時送信 |
| `zip_code` | 郵便番号 | `zip` に送信 |
| `industry` | 業種 | `industry` に送信 |
| `employee_count` | 従業員数 | `numberofemployees` に送信 |
| `notes` | 備考 | `description` に送信 |
| `representative` | 代表者名 | `Contact` 登録で `lastname`/`firstname` に分割して送信 |

> **備考**: 空文字 / None の値は送信前に削除され、HubSpot APIリクエストに含めない。

---

## 3. 出力／挙動

### 3-1. 成功時
- `true` を返す
- ログに `企業登録成功: ... (ID:...)` を出力

### 3-2. 重複検出
- ドメインで重複：`_search_company_by_domain()` で確認
- 会社名で重複：`_search_company_by_name()` で確認
- 名前の正規化（法人格除去）での重複もチェック
- 重複時は登録せず `false` を返す

### 3-3. 失敗時（主なパターン）
- **電話番号フォーマットエラー**（HubSpot API が `INVALID_PHONE_NUMBER` を返す）
  - まず電話番号を除外して再送登録
  - それでも失敗する場合は `false` を返し、NGリストに記録
- **HubSpot APIエラー**（400/409 以外）
  - ログ出力して `false` を返す
- **例外発生**
  - 例外をログ出力し `false` を返す

## 4. 既知の失敗ケース（Known Issues）

### 4-1. 郵便番号が電話番号欄に紛れ込む
- 収集ロジック上、`電話番号`欄に `004-0821` のような **郵便番号形式（`\d{3}-\d{4}`）** が入ることがある
- 現在は `_normalize_phone()` で **この形式を空文字扱い**にして送信しないようにしている

### 4-2. 電話番号フォーマット不正（+81 形式でない）
- `03-1234-5678` → `+81312345678` へ正規化
- `090-1234-5678` → `+819012345678` へ正規化
- `+81...` ならそのまま使用

### 4-3. 内線番号の扱い
- `～内線123` / `ext123` / `x123` などを検知し、`+81... ext 123` の形式へ変換

## 5. テストケース（追加推奨）

以下のユニットテストを用意しておくと、コード修正時のリグレッションを防げる。

- `_normalize_phone()` が以下を正しく変換する
  - `03-1234-5678` → `+81312345678`
  - `090-1234-5678` → `+819012345678`
  - `+81-90-1234-5678` → `+819012345678`
  - `03-1234-5678 内線123` → `+81312345678 ext 123`
  - `004-0821` → `` (空)
  - `061-1405` → `` (空)

- `register_company()` が「電話フォーマットエラーで再送するロジック」を実際に通ること（モックHubSpotで再現）

**テストコード例**
- `tests/test_hubspot_agent.py` を参照

**テスト実行方法**
```bash
python -m unittest -q tests.test_hubspot_agent
```

## 6. 運用ルール（推奨）

### 6-1. 仕様変更時の手順
1. このMDを更新
2. 該当エージェントにテストを追加/更新
3. `python -m pytest` などで自動化テストを通す
4. リリース

### 6-2. バグを見つけたら
1. このMDの「既知の失敗ケース」に追記
2. コードを修正し、テストを追加
3. その修正をレビュー・マージ
