# Agent: スクレイパーエージェント（scraper_agent.py）

**目的**
- 企業の公式サイト（トップページ / 会社概要ページ）をスクレイピングし、リスト登録に必要な企業情報を取得する。
- 取得情報を整形・検証し、信頼度スコアを付与する。

---

## 1. 役割（責務）

### ✅ やること
- 企業サイトの「会社概要」的なページから以下の情報を抜き出す
  - 会社名、住所、電話、代表者、従業員数、業種、郵便番号、都道府県
- 取得した情報に対してバリデーションを実施し、信頼度スコアを付与（`_confidence`）
- 企業ページとして不十分（会社概要項目が不足・日本語ページでないなど）の場合は候補から除外

### ❌ やらないこと
- 検索エンジンへのクエリ発行（`search_agent` が担当）
- ランク付け（`rank_agent` が担当）
- HubSpot登録（`hubspot_agent` が担当）

---

## 2. 主な機能

### 2-1. `scrape_company_info(url: str, ...) -> dict`
- `url`（企業サイトURLまたは媒体記事URL）から企業情報を抽出する。
- 「媒体記事ページ」経由の場合は、記事中から企業URLを抽出してそこをスクレイピングする。
- 取得情報は以下のキーを含む辞書で返す（不足分は空文字）:
  - `company_name`, `company_url`, `representative`, `address`, `zip_code`, `prefecture`, `phone`, `employee_count`, `industry`, `source_url` など
- 最低限の必須要件を満たさない場合は `company_url` を空にして返すことで後続処理をスキップさせる。

### 2-2. `check_company_fields(all_text: str) -> tuple[bool, list[str]]`
- 取得したページテキストから「企業サイトとして妥当か」を判定する。
- 判定基準（5項目中3つ以上が成立）:
  1. 法人格（株式会社/合同会社 など）
  2. 代表者（代表取締役 / 代表者 / 社長 など）
  3. 事業内容（事業内容 / 会社情報 / About など）
  4. 住所（〒または都道府県+市区町村）
  5. TEL（電話番号）
- 以下の追加チェックを導入:
  - **従業員数の記載がない場合は即除外**（人数がわからないとターゲット精度が落ちるため）
  - **IR / 上場情報（東証グロース・スタンダード等）が含まれている場合は除外**

- **項目名はページごとに異なるが、ラベルに依存せずに抽出している**
  - `scraper_agent` では各フィールドを「テキスト全体から正規表現で抽出」するため、
    ラベルが「連絡先」「TEL」「電話番号」など異なっていても問題ない。
  - 例えば代表者は `代表取締役` / `代表者` / `社長` などのパターン、
    電話は `TEL: 03-xxxx-xxxx` や `電話番号 03-xxxx-xxxx` などを網羅している。
  - したがって「項目名が違うから取れない」ではなく「該当情報がテキストに含まれていない」場合に未取得となる。

### 2-3. 取得・正規化の補助処理
- `extract_employee_count(text)` で「従業員数: XX名」等から数字を抽出
- `extract_address_parts(text)` で郵便番号 / 都道府県 / 住所を抽出
- `validate_company_info(info)` で電話番号/従業員数の妥当性を検証し、信頼度を計算

---

## 3. 運用上の注意

- `check_company_fields()` による「日本語ページ判定」は、ひらがな・カタカナが一定量含まれない場合に除外するため、英語サイトやレンダリング依存サイトは除外されやすい。
- 事業内容記載が弱いサイト（コーポレートサイトではなく、サービスサイトや採用サイト等）は除外される可能性が高い。
- `従業員数` の抽出パターンは `EMPLOYEE_PATTERNS` で定義されているため、記載フォーマットが異なる場合はここを拡張する。
