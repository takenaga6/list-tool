# Agent: 検索エージェント（search_agent.py）

**目的**
- キーワードから検索エンジン（DuckDuckGo）を使い、企業の公式サイトに絞ったURL候補を抽出する。
- ノイズ（ニュース記事・メディア・求人・ECなど）の混入を防ぎ、精度を高める。

---

## 1. 役割（責務）

### ✅ やること
- キーワード（例: "代表取締役 取り上げられました"）に対して検索を実行
- 取得結果から「企業公式サイトっぽいURL」を判定して戻す
- 除外ドメイン・海外ドメイン・メディアドメインはフィルタリングする

### ❌ やらないこと
- 企業情報（電話・住所・業種など）の抽出
- 検索キーワードの生成（rank_agent / keyword_agent が担当）

---

## 2. 入力（引数）

- `query: str` ・・・ 検索キーワード（例: "整体 代表取締役 取材"）
- `tbs: str` ・・・ 検索期間フィルタ（例: `qdr:m` など、内部で DuckDuckGo 形式に変換）
- `num: int` ・・・ 取得する候補数（デフォルト 10）

---

## 3. 出力（戻り値）

`list[dict]` 形式で、各アイテムは以下を含む

- `url`：候補URL
- `title`：検索結果タイトル
- `snippet`：検索結果スニペット
- `search_query`：元の検索語
- `is_media_page`：メディア（媒体）判定フラグ（現状常に False）
- `media_domain`：媒体ドメイン（媒体クエリ検出時）
- `file_type`：`pdf`/`xlsx`/`docx`など

---

## 4. 精度向上のための自動除外・フィルタリング

`search_agent.py` では以下の仕組みでノイズを取り除き、精度を上げている。

### 4-1. ドメイン除外（`EXCLUDE_DOMAINS`）
- SNS、EC、ニュース、企業DBなどのドメインを列挙してあり、該当する URL はすべて除外される。
- 追加が必要な場合は、この配列にドメインを追加する。

### 4-2. ドメインキーワード除外（`NG_DOMAIN_KEYWORDS`）
- `news`, `media`, `review`, `catalog` など、企業HPではない可能性が高いキーワードを含むドメインを除外。
- 追加すれば、同じ意味合いのドメインをまとめて除外できる。

### 4-3. 海外ドメイン除外（`FOREIGN_TLDS`）
- 日本企業ターゲットなので `.us` や `.com.au` などの海外TLDは除外される。

### 4-4. メディア名クエリ（`MEDIA_NAME_TO_DOMAIN`）
- キーワードに媒体名が含まれている場合、媒体ドメインを認識して除外する。
- `config.py` の `MEDIA_NAME_TO_DOMAIN` を更新すると追加できる。

### 4-5. 企業公式サイトっぽいチェック（`_looks_like_company_url()`）
- **ここが最も精度に直結する箇所**
- 判定基準は以下の通り
  1. `.co.jp` なら無条件 OK
  2. `.com/.net/.jp` なら「法人格」「会社概要系URL」「企業HPっぽいワード」 のどれか1つ以上が必要
  3. その他TLDは「追加シグナル 2つ以上」で OK

### 4-6. どのURLを落としているか（調査用ログ）
- 判定NGでフィルタされた URL は `output/rejected_search_urls.csv` に保存されるようになりました。
- どのURLが「シグナル不足」で落ちたかを CSV で確認し、精度改善候補を見つける用途に使えます。
- ファイルは実行ごとに追記されるため、不要になったら削除してリセットしてください。

---

## 5. フィードバック・学習ループ（テレアポログとの連携）

- `search_agent` では「どのURLを落としたか」を `output/rejected_search_urls.csv` に残し、
  それが精度改善（フィルタ追加・判定ルール調整）への素材になります。
- また、`rank_agent` が判定した NG/アポ獲得の結果は `keyword_agent` の学習（`keyword_stats.json`）に反映されます。
- テレアポ・商談ログ（`output/feedback.csv`）を使って「実際にアポが取れたか/NG理由」を定量化し、
  `search_agent` の除外ルールや `rank_agent` のシグナル重みを調整する拡張も可能です。

---

## 6. 精度改善のためにできること（次のステップ）

### 5-1. 追加シグナルの拡張
- `CORP_KEYWORDS`（スニペット/タイトルに含まれる企業HPっぽいワード）を増やす
  - 例: `お問い合わせ`, `アクセス`, `製品情報`, `サービス概要` など
- `COMPANY_PATHS`（URLパスに含まれると企業ページとみなすセグメント）を増やす
  - 例: `/service`, `/business`, `/service-info` など

### 5-2. 除外ドメインの自動学習の改善
- `load_exclude_list_csv()` で読み込む除外リストに「今までエラーになったドメイン」を追加していくと精度向上につながる

### 5-3. テスト追加（自動検証）
- `_looks_like_company_url()` のテストを用意し、さまざまなパターンを回帰検証する。
  - 例: `www.example.com/about` は企業ページとして通る（期待）
  - 例: `www.example.com/news` は通らない（期待）
